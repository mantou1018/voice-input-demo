继续补充后，不想再说了

下面是 3 个可视化方案。每个手机稿都模拟同一时刻：用户已识别到“北京”，点了“继续补充”，但进入补充录音后改主意，想直接报名。

方案 A

左上角返回即回确认态

不新增按钮。用户在补充录音页点返回，直接回到确认页，再决定报名或继续改。

补充录音中 最省改动
请确认信息
用户此刻点返回
我今年 岁
我的手机号是
我的意向城市是 北京
我的意向职位是
北京
点返回后，直接回到“确认并报名”页面。
用户结果：回到确认态，底部重新看到 `继续补充` 和 `确认并报名`。
优点：最自然 风险：返回含义略不明确
方案 B

底部直接给“返回确认”

补充录音态的底部操作改成更明确的三选二结构，用户不用猜返回键含义。

补充录音中 语义最清楚
请确认信息
用户不想继续说时可直接回确认
我今年 岁
我的手机号是
我的意向城市是 北京
我的意向职位是
北京
不想继续补充了，可以直接返回确认。
用户结果：在当前页就能明确选择“继续补”还是“回去报名”。
优点：不易误解 代价：底部按钮要改
方案 C

补充前先给二次选择

用户点“继续补充”后，不立刻开录，而是先给一个轻选择页。适合希望路径更稳的版本。

继续补充前 最稳妥
还要补充信息吗
现在可以直接报名，也可以继续补充
已识别：北京
未补：年龄 / 手机号 / 职位
你可以继续补充，也可以先回去确认报名。
用户结果：不会被动进入补充录音，但比现在多一步。
优点：不会误入录音 代价：流程更长
